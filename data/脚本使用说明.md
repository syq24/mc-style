# data 目录脚本使用说明

本文档汇总 data 目录下主要脚本的用途、运行命令与参数含义。

适用环境建议：
- Linux
- conda 环境：wan210
- 当前项目根目录：/root/autodl-tmp/mc-style

建议先执行：
  conda activate wan210
  cd /root/autodl-tmp/mc-style

一、OpenVidHD 数据筛选脚本
文件：data/filter_openvidhd_buckets.py

1) 作用
- 从 OpenVidHD.csv 中按 caption 关键词筛出两类样本：
  - people
  - generic_scene
- 过滤负样本关键词（如 anime、cartoon、watermark 等）
- 按质量分排序并可限制每个桶最大数量

2) 典型运行指令
- 使用默认参数：
  python data/filter_openvidhd_buckets.py

- 指定输入输出与时长范围：
  python data/filter_openvidhd_buckets.py \
    --input /root/autodl-tmp/mc-style/data/OpenVidHD.csv \
    --output /root/autodl-tmp/mc-style/data/OpenVidHD_30k_people_scene.csv \
    --min-seconds 4 \
    --max-seconds 12 \
    --max-people 18000 \
    --max-scene 12000

3) 参数说明
- --input
  输入 CSV 路径。需至少包含 video、caption、seconds 列。
- --output
  输出筛选后 CSV 路径。
- --min-seconds
  最小时长阈值（秒），仅保留 seconds >= 该值。
- --max-seconds
  最大时长阈值（秒），仅保留 seconds <= 该值。
- --max-people
  people 桶最多保留多少条。默认不限制。
- --max-scene
  generic_scene 桶最多保留多少条。默认不限制。

4) 排序逻辑
- 脚本会计算 quality_score：
  - aesthetic score 排名权重 0.5
  - temporal consistency score 排名权重 0.35
  - motion score 排名权重 0.15
- 然后按 quality_score、aesthetic score、seconds 进行排序。

二、OpenVidHD 分片映射脚本
文件：data/map_openvidhd_parts.py

1) 作用
- 将筛选后 CSV 中的 video 文件名，映射到 OpenVidHD 的 part/zip 分片。
- 生成下载和调度所需的多个文件：
  - *_summary.csv
  - *_mapped.csv
  - *_zip_list.txt
  - *_missing.txt

2) 典型运行指令
- 默认参数：
  python data/map_openvidhd_parts.py

- 指定输入与输出前缀：
  python data/map_openvidhd_parts.py \
    --input /root/autodl-tmp/mc-style/data/OpenVidHD_30k_people_scene.csv \
    --json /root/autodl-tmp/mc-style/data/OpenVidHD.json \
    --output-prefix /root/autodl-tmp/mc-style/data/OpenVidHD_30k_people_scene_parts

3) 参数说明
- --input
  经过筛选后的 CSV，必须有 video 列。
- --json
  OpenVidHD 的映射 JSON 文件（part 到文件列表）。
- --output-prefix
  输出文件前缀。脚本会自动拼接不同后缀生成 4 个结果文件。

4) 输出文件含义
- summary.csv
  每个 part 命中数量统计，通常用于按收益排序下载。
- mapped.csv
  每条样本对应的 part_name、zip_name 等明细。
- zip_list.txt
  需要下载的 zip 列表。
- missing.txt
  在 JSON 映射中找不到的视频名。

三、OpenVidHD 顺序下载与清洗脚本
文件：data/download_openvidhd_parts.py

1) 作用
- 按 summary 排名区间下载 OpenVidHD 分片。
- 每次处理一个 part：下载 -> 组装（必要时）-> 解压 -> 拷贝目标视频 -> 清理临时文件。
- 支持分片拼接格式（例如 part_aa/part_ab）。
- 支持下载重试和下载中断续传（基于 .part 文件 + HTTP Range）。

2) 常用运行指令
- 下载前 10 个高收益 part：
  python data/download_openvidhd_parts.py \
    --rank-range 1-10 \
    --endpoint https://hf-mirror.com

- 下载 11 到 20：
  python data/download_openvidhd_parts.py \
    --rank-range 11-20 \
    --endpoint https://hf-mirror.com

- 先做 dry-run 检查：
  python data/download_openvidhd_parts.py \
    --rank-range 1-10 \
    --endpoint https://hf-mirror.com \
    --dry-run

3) 参数说明（核心）
- --summary-csv
  来自 map 脚本的汇总文件，默认 OpenVidHD_30k_people_scene_parts_summary.csv。
- --mapped-csv
  来自 map 脚本的映射明细，默认 OpenVidHD_30k_people_scene_parts_mapped.csv。
- --rank-range
  1-based 排名区间，格式例如 1-10、11-20 或单值 5。
- --content-dir
  保留的视频输出目录。
- --work-dir
  下载、组装、解压临时目录。
- --repo-id
  Hugging Face 数据集仓库名，默认 nkp37/OpenVid-1M。
- --repo-subdir
  数据集内子目录，默认 OpenVidHD。
- --endpoint
  下载基地址，默认读取 HF_ENDPOINT 环境变量，否则 https://hf-mirror.com。
- --revision
  数据集版本，默认 main。
- --split-start
  当 part 编号 >= 该值时，按拆分文件规则处理。
- --split-suffixes
  拆分后缀列表，默认 aa,ab。
- --log-dir
  运行日志输出目录。
- --connect-timeout
  HTTP 连接超时秒数。
- --read-timeout
  HTTP 读取超时秒数（流式下载尤其关键）。
- --max-retries
  单个远程文件最大重试次数。
- --retry-backoff
  失败后退避基数秒；实际等待为 backoff * attempt。
- --overwrite-existing
  目标视频已存在时是否覆盖。
- --force-redownload
  忽略本地缓存，强制重下。
- --dry-run
  仅解析目标 part 和远程文件，不做真实下载与解压。

4) 日志与结果
- 每次运行会在 log-dir 下创建 download_run_时间戳 目录。
- parts_log.csv 记录每个 part 的状态、缺失数量、错误信息。
- 若有缺失视频，会写 partX_missing.txt。

四、OpenVidHD tmux 后台启动脚本
文件：data/run_download_openvidhd_parts_tmux.sh

1) 作用
- 用 tmux 后台启动下载任务，避免断开 SSH/VS Code 后任务停止。
- 自动写控制台日志到 data/logs。

2) 用法
- 基本格式：
  bash data/run_download_openvidhd_parts_tmux.sh 排名区间 会话名 其余参数...

- 示例1（默认会话名）：
  bash data/run_download_openvidhd_parts_tmux.sh 1-10

- 示例2（指定会话名 + endpoint）：
  bash data/run_download_openvidhd_parts_tmux.sh 11-20 openvid_11_20 --endpoint https://hf-mirror.com

- 示例3（附加下载超参）：
  bash data/run_download_openvidhd_parts_tmux.sh 1-10 openvid_fast \
    --endpoint https://hf-mirror.com \
    --max-retries 10 \
    --read-timeout 900

3) 位置参数说明
- 第1个参数：rank-range，例如 1-10。
- 第2个参数：tmux session 名，不传则自动生成。
- 后续参数：会原样透传给 download_openvidhd_parts.py。

4) 观察任务
- 连接会话：
  tmux attach -t 会话名
- 查看日志：
  tail -f data/logs/对应日志文件.log

五、WikiArt 本地 parquet 转图片脚本
文件：data/export_wikiart_from_hf.py

1) 作用
- 将本地 raw 目录中的 parquet 样本导出为 images 与 metadata。
- 支持输出 jpg/png。
- metadata 支持 csv，可选 txt。

2) 典型运行指令
- 默认路径直接转换：
  python data/export_wikiart_from_hf.py

- 限制样本量做冒烟测试：
  python data/export_wikiart_from_hf.py \
    --max-parquet-files 2 \
    --max-samples 1000

- 跳过已存在图片并输出 txt：
  python data/export_wikiart_from_hf.py \
    --skip-existing-images \
    --write-txt

3) 参数说明
- --root-dir
  WikiArt 根目录，内部应包含 raw、images、metadata。
- --raw-glob
  在 raw 下匹配 parquet 的 glob，默认 data/*.parquet。
- --max-parquet-files
  最多处理多少个 parquet 文件，默认不限制。
- --max-samples
  全局最多处理多少样本，默认不限制。
- --batch-size
  Arrow 迭代批大小，影响吞吐和内存占用。
- --image-format
  输出图片格式，jpg 或 png。
- --jpeg-quality
  image-format=jpg 时的质量参数。
- --skip-existing-images
  若目标图片已存在则不重写（但仍写入 metadata 行）。
- --write-txt
  额外输出 txt 清单。

4) 输出说明
- 图片目录：root-dir/images
- 元数据：root-dir/metadata/wikiart_metadata.csv
- 可选文本清单：root-dir/metadata/wikiart_metadata.txt

六、策略B图片缩放脚本
文件：data/resize_images_strategy_b.py

1) 作用
- 对目录内图片递归遍历，按面积阈值缩放：
  width * height <= max_pixels
- 保持宽高比，不放大小图。
- 支持原地覆盖或输出到新目录。

2) 典型运行指令
- 预览不写盘：
  python data/resize_images_strategy_b.py \
    --input-dir /root/autodl-tmp/mc-style/data/style_images/wikiart/images \
    --max-pixels 399360 \
    --dry-run

- 原地处理：
  python data/resize_images_strategy_b.py \
    --input-dir /root/autodl-tmp/mc-style/data/style_images/wikiart/images \
    --max-pixels 399360

- 输出到新目录：
  python data/resize_images_strategy_b.py \
    --input-dir /root/autodl-tmp/mc-style/data/style_images/wikiart/images \
    --output-dir /root/autodl-tmp/mc-style/data/style_images/wikiart/images_b399360 \
    --max-pixels 399360

3) 参数说明
- --input-dir
  输入图片根目录，递归扫描。
- --output-dir
  输出根目录。不传则原地改写。
- --max-pixels
  最大像素面积阈值。策略B推荐 399360。
- --jpeg-quality
  JPG 输出质量。
- --png-compress-level
  PNG 压缩级别 0 到 9。
- --keep-mtime
  原地写回时保留原修改时间。
- --dry-run
  仅打印将会缩放的文件，不实际写入。

4) 统计输出
- 扫描总数、实际缩放数、跳过数、错误数。

十、常见问题

1) 下载中断后是否可续传
- 可以。download_openvidhd_parts.py 下载阶段基于 .part 文件和 HTTP Range 续传。

2) 为什么有些 part 会出现缺失
- 可能是映射文件与压缩包内容存在差异，脚本会在日志目录输出 missing 列表。

3) 我应该调哪些超参数
- 网络不稳：提高 --max-retries 与 --read-timeout。
- 想更保守：先 --dry-run，再小区间 --rank-range 1-2 实跑。
- 避免覆盖已有结果：不要加 --overwrite-existing。
